# Práctica: Explicación de GitHub y Azure

## Descripción

Esta práctica tiene como objetivo introducir los conceptos básicos de GitHub y Azure, mostrando cómo se pueden utilizar juntos para gestionar código fuente y desplegar aplicaciones en la nube.

## Objetivos

- Comprender los conceptos fundamentales de GitHub.
- Aprender a crear y gestionar repositorios.
- Realizar operaciones básicas: commit, push, pull y manejo de ramas.
- Introducción a Azure y sus servicios principales.
- Desplegar una aplicación desde GitHub a Azure.

## Requisitos Previos

- Tener una cuenta en [GitHub](https://github.com/).
- Tener una cuenta en [Azure](https://portal.azure.com/).
- Git instalado en tu equipo.
- Visual Studio Code o cualquier editor de código.

## Pasos de la Práctica

### 1. Configuración de GitHub

1. Crea un nuevo repositorio en GitHub.
2. Clona el repositorio en tu equipo local.
3. Agrega un archivo `README.md` y realiza tu primer commit.
4. Sube los cambios al repositorio remoto.

### 2. Introducción a Azure

1. Accede al portal de Azure.
2. Crea un recurso (por ejemplo, una Web App).
3. Configura la integración continua con GitHub.

### 3. Despliegue de la Aplicación

1. Realiza cambios en tu repositorio de GitHub.
2. Verifica que Azure despliegue automáticamente los cambios.
